# Mudita LMS — Course Blueprints: Ages 6–8 (Young Creators)

> **Learner profile:** Energetic beginners ready for short structured tasks, guided creativity, simple digital tools, and beginner problem-solving.
> **Session length:** 20–25 minutes per lesson. Screen-based activities mixed with hands-on tasks.

---

## Course 6: Coding Adventures with Blocks

| Field | Value |
|-------|-------|
| **Age Group** | 6–8 (Young Creators) |
| **Type** | FREE |
| **Difficulty** | Beginner |
| **Duration** | 4 weeks (9 lessons, ~20 min each) |
| **Prerequisite** | None (Little Coders Unplugged recommended) |
| **Course Promise** | Your child will write their first real code using block-based programming — creating animations, solving puzzles, and building a mini game. |
| **Parent-Facing Value** | Block coding (used on platforms like Code.org and Scratch Jr) is the proven on-ramp to programming. Your child will build sequencing, loop, and event-driven thinking — all through playful projects that feel like games, not lessons. |
| **Completion Criteria** | Complete all 9 lessons + publish a Mini Maze game |
| **Badge** | "Block Coder" — awarded on completion |
| **Certificate** | "Young Creator: Coding Adventures with Blocks" |

### Learner Outcomes
1. Explain that code is a set of instructions a computer follows
2. Sequence blocks to create a simple animation
3. Use a loop block to repeat actions
4. Identify and fix a bug in a simple program
5. Build and share a playable mini maze game

### Module 1: Code Is Instructions (Lessons 1–3)

**Lesson 1: Give the Computer Steps**
- Objective: Understand that code tells a computer exactly what to do, step by step
- Summary: Just like a robot follows commands, computers follow code. Children arrange blocks to make a character walk, jump, and wave — seeing instantly how each block produces an action.
- Activity: Drag 5 action blocks into sequence and run the program. Change the order and observe what happens.
- Assessment: Child correctly sequences at least 3 blocks to produce an intended outcome

**Lesson 2: Find the Bug**
- Objective: Identify an error in a simple block program and correct it
- Summary: A "bug" is when code does something you didn't want. Children review a broken program where a character goes the wrong way, and fix it by swapping or removing a block.
- Activity: Fix 3 broken programs — each has exactly one wrong block to find and replace
- Assessment: Child successfully fixes all 3 programs

**Lesson 3: My First Animation**
- Objective: Create an original animated scene using at least 4 action blocks
- Summary: Children use what they've learned to make their own animation — a character that moves, speaks, and changes. Their first real independent program.
- Activity: Create an animation with a character doing at least 3 different actions in sequence
- Assessment: Program runs without bugs and produces the intended sequence

### Module 2: Loops and Patterns (Lessons 4–6)

**Lesson 4: Repeat It**
- Objective: Use a loop block to repeat an action multiple times
- Summary: Instead of writing "jump, jump, jump, jump," coders write "repeat 4 times: jump." Children discover loops and see how they make programs shorter and smarter.
- Activity: Rewrite a 8-block program using a repeat loop — make it do the same thing with fewer blocks
- Assessment: Child creates a working loop that repeats an action at least 3 times

**Lesson 5: Dance Code**
- Objective: Combine loops with sequences to create a repeating dance pattern
- Summary: Children program a character to perform a dance — a series of moves that repeat. They experiment with different loop counts to change the dance length.
- Activity: Create a 3-move dance that loops at least twice using a repeat block
- Assessment: Dance runs correctly with a visible repeating pattern

**Lesson 6: Forever Loops**
- Objective: Understand the difference between a counted loop and a forever/continuous loop
- Summary: Some loops run a set number of times; others run forever (until you stop them). Children explore "forever" loops for things like a spinning windmill or bouncing ball.
- Activity: Create a forever loop animation (spinning, bouncing, or wiggling character)
- Assessment: Child can explain the difference between "repeat 5" and "forever"

### Module 3: Stories and Games (Lessons 7–9)

**Lesson 7: Animate a Scene**
- Objective: Use events (click, key press) to trigger actions in a scene
- Summary: What if the character only moves when you press a key? Children add event blocks — "when green flag clicked," "when key pressed" — to make their programs interactive.
- Activity: Build a scene where pressing the spacebar makes something happen (character jumps, star appears, sound plays)
- Assessment: At least one event block triggers an action correctly

**Lesson 8: Build a Mini Maze**
- Objective: Combine movement blocks and events to create a navigable maze
- Summary: Children start building a mini maze game — a character that moves left/right/up/down when arrow keys are pressed, trying to reach a goal.
- Activity: Program arrow key controls for a character in a simple 2-wall maze
- Assessment: All 4 directional keys move the character in the correct direction

**Lesson 9: Publish Your Game (Final Project)**
- Objective: Complete, test, debug, and share the mini maze game
- Summary: Children finish and polish their maze game — test it, fix any bugs, give it a title, and share it. This is their first published project.
- Activity: Complete the maze with a start point, goal, at least 2 walls, and working controls. Test with a friend or family member.
- Assessment: Game is playable from start to goal, controls work, no game-breaking bugs

## Course 7: Science Detectives

| Field | Value |
|-------|-------|
| **Age Group** | 6–8 (Young Creators) |
| **Type** | PAID |
| **Difficulty** | Beginner |
| **Duration** | 4 weeks (9 lessons, ~20 min each) |
| **Prerequisite** | None |
| **Course Promise** | Your child will think and act like a real scientist — asking questions, running experiments, and explaining what they discover about matter, weather, and living things. |
| **Parent-Facing Value** | Grounded in the scientific method used by organizations like NASA and National Geographic Kids, this course builds inquiry habits — forming hypotheses, observing carefully, and explaining findings — that strengthen every school subject. |
| **Completion Criteria** | Complete all 9 lessons + present a Science Detective Report |
| **Badge** | "Science Detective" — awarded on completion |
| **Certificate** | "Young Creator: Science Detectives" |

### Learner Outcomes
1. Describe what scientists do and why asking questions matters
2. Make a prediction (hypothesis) before an experiment
3. Classify matter as solid, liquid, or gas
4. Explain what plants need to survive and grow
5. Describe at least two different animal habitats

### Module 1: Ask and Observe (Lessons 1–3)

**Lesson 1: What Do Scientists Do?**
- Objective: Understand that scientists observe, question, test, and share
- Summary: Scientists aren't just people in white coats — they're curious people who notice things, ask "why?", and find out. Children explore the 4 steps of simple scientific inquiry.
- Activity: Watch a nature observation video, then list 3 things observed and write 1 question about each
- Assessment: Child produces 3 observations and at least 1 "I wonder…" question

**Lesson 2: Make a Prediction**
- Objective: Form a simple hypothesis before testing
- Summary: A prediction isn't a guess — it's an informed idea. "I think the ice will melt faster in warm water because..." Children practice making "I think… because…" statements.
- Activity: Given 3 experiment scenarios, write a prediction for each using the sentence frame
- Assessment: All 3 predictions follow the "I think… because…" format

**Lesson 3: Test It Out**
- Objective: Conduct a simple experiment and record what happened
- Summary: Children run a simple home experiment (e.g., which paper towel absorbs more water?) and record their prediction, result, and whether it matched.
- Activity: Absorption test — test 2 different paper towels, record prediction vs result
- Assessment: Child records a prediction, result, and a one-sentence conclusion

### Module 2: Matter and Weather (Lessons 4–6)

**Lesson 4: Solid, Liquid, Gas**
- Objective: Classify matter into the three main states and describe each
- Summary: Everything around us is matter — and matter comes in three forms. Ice is solid, water is liquid, steam is gas. Children explore examples and learn what makes each state different.
- Activity: Sort 12 picture cards into solid, liquid, gas. Then: what happens to ice when it warms up?
- Assessment: Child correctly sorts at least 10 of 12 cards and describes one state change

**Lesson 5: Clouds, Rain, Wind**
- Objective: Explain the water cycle in simple terms and identify types of clouds
- Summary: Where does rain come from? The water cycle! Sun heats water → it rises as vapor → forms clouds → falls as rain. Children trace the cycle and match cloud types to weather.
- Activity: Draw and label the water cycle with 4 steps; match 3 cloud types to their weather (sunny, rainy, stormy)
- Assessment: Water cycle has all 4 steps labeled; cloud matching is correct

**Lesson 6: Weather Detective**
- Objective: Record and describe daily weather observations
- Summary: Meteorologists study weather every day. Children become weather detectives — observing, measuring (warm/cold), and recording the weather for 3 days.
- Activity: 3-day weather journal: temperature description, cloud type, precipitation, wind
- Assessment: Journal completed for 3 days with at least 3 observations per day

### Module 3: Living Things (Lessons 7–9)

**Lesson 7: Plants Need What?**
- Objective: Identify what plants need to survive: sunlight, water, air, soil
- Summary: Plants are alive — and like all living things, they have needs. Children explore what happens when a plant doesn't get water, light, or soil, and set up a simple plant observation.
- Activity: Set up a "plant needs" experiment — grow two small seedlings, one in light and one in dark, and observe for a week
- Assessment: Child can name 4 plant needs and predict what will happen in the experiment

**Lesson 8: Habitats Around Us**
- Objective: Describe 3 different animal habitats and the animals that live there
- Summary: Animals live in habitats that match their needs — a polar bear needs cold, a camel needs dry. Children explore forest, ocean, and desert habitats.
- Activity: Habitat matching card game — match 9 animals to their correct habitat (forest, ocean, desert)
- Assessment: Child correctly matches at least 7 of 9 animals

**Lesson 9: Science Detective Report (Final Project)**
- Objective: Present findings from the plant experiment and summarize science learning
- Summary: Children revisit their plant experiment, record final observations, and present a short "Science Detective Report" covering what they asked, what they tried, and what they found.
- Activity: Complete a 4-section report: Question → Prediction → Experiment → Result. Present to family.
- Assessment: All 4 sections completed; child can verbally explain what they discovered
