> **Session length:** 20–25 minutes per lesson. Screen-based activities mixed with hands-on tasks.

## Course 6: Coding Adventures with Blocks

| Field | Value |
|-------|-------|
| **Age Group** | 6–8 (Young Creators) |
| **Type** | FREE |
| **Difficulty** | Beginner |
| **Duration** | 4 weeks (9 lessons, ~20 min each) |
| **Prerequisite** | None (Little Coders Unplugged recommended) |
| **Course Promise** | Your child will write their first real code using block-based programming — creating animations, solving puzzles, and building a mini game. |
| **Parent-Facing Value** | Block coding (used on platforms like Code.org and Scratch Jr) is the proven on-ramp to programming. Your child will build sequencing, loop, and event-driven thinking — all through playful projects that feel like games, not lessons. |

### Learner Outcomes
1. Explain that code is a set of instructions a computer follows
2. Sequence blocks to create a simple animation
3. Use a loop block to repeat actions
4. Identify and fix a bug in a simple program
5. Build and share a playable mini maze game

### Module 1: Code Is Instructions (Lessons 1–3)

**Lesson 1: Give the Computer Steps**
- Objective: Understand that code tells a computer exactly what to do, step by step
- Summary: Just like a robot follows commands, computers follow code. Children arrange blocks to make a character walk, jump, and wave — seeing instantly how each block produces an action.
- Activity: Drag 5 action blocks into sequence and run the program. Change the order and observe what happens.
- Assessment: Child correctly sequences at least 3 blocks to produce an intended outcome

**Lesson 2: Find the Bug**
- Objective: Identify an error in a simple block program and correct it
- Summary: A "bug" is when code does something you didn't want. Children review a broken program where a character goes the wrong way, and fix it by swapping or removing a block.
- Activity: Fix 3 broken programs — each has exactly one wrong block to find and replace
- Assessment: Child successfully fixes all 3 programs

**Lesson 3: My First Animation**
- Objective: Create an original animated scene using at least 4 action blocks
- Summary: Children use what they've learned to make their own animation — a character that moves, speaks, and changes. Their first real independent program.
- Activity: Create an animation with a character doing at least 3 different actions in sequence
- Assessment: Program runs without bugs and produces the intended sequence

### Module 2: Loops and Patterns (Lessons 4–6)

**Lesson 4: Repeat It**
- Objective: Use a loop block to repeat an action multiple times
- Summary: Instead of writing "jump, jump, jump, jump," coders write "repeat 4 times: jump." Children discover loops and see how they make programs shorter and smarter.
- Activity: Rewrite a 8-block program using a repeat loop — make it do the same thing with fewer blocks
- Assessment: Child creates a working loop that repeats an action at least 3 times

**Lesson 5: Dance Code**
- Objective: Combine loops with sequences to create a repeating dance pattern
- Summary: Children program a character to perform a dance — a series of moves that repeat. They experiment with different loop counts to change the dance length.
- Activity: Create a 3-move dance that loops at least twice using a repeat block
- Assessment: Dance runs correctly with a visible repeating pattern

**Lesson 6: Forever Loops**
- Objective: Understand the difference between a counted loop and a forever/continuous loop
- Summary: Some loops run a set number of times; others run forever (until you stop them). Children explore "forever" loops for things like a spinning windmill or bouncing ball.
- Activity: Create a forever loop animation (spinning, bouncing, or wiggling character)
- Assessment: Child can explain the difference between "repeat 5" and "forever"

### Module 3: Stories and Games (Lessons 7–9)

**Lesson 7: Animate a Scene**
- Objective: Use events (click, key press) to trigger actions in a scene
- Summary: What if the character only moves when you press a key? Children add event blocks — "when green flag clicked," "when key pressed" — to make their programs interactive.
- Activity: Build a scene where pressing the spacebar makes something happen (character jumps, star appears, sound plays)
- Assessment: At least one event block triggers an action correctly

**Lesson 8: Build a Mini Maze**
- Objective: Combine movement blocks and events to create a navigable maze
- Summary: Children start building a mini maze game — a character that moves left/right/up/down when arrow keys are pressed, trying to reach a goal.
- Activity: Program arrow key controls for a character in a simple 2-wall maze
- Assessment: All 4 directional keys move the character in the correct direction

**Lesson 9: Publish Your Game (Final Project)**
- Objective: Complete, test, debug, and share the mini maze game
- Summary: Children finish and polish their maze game — test it, fix any bugs, give it a title, and share it. This is their first published project.
- Activity: Complete the maze with a start point, goal, at least 2 walls, and working controls. Test with a friend or family member.
- Assessment: Game is playable from start to goal, controls work, no game-breaking bugs
