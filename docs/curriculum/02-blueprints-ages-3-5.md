# Mudita LMS — Course Blueprints: Ages 3–5 (Early Explorers)

> Learner profile: Curious, sensory, playful learners who learn best through movement, stories, repetition, songs, role-play, and hands-on discovery.
> Session length: 15–20 minutes per lesson. Short, movement-rich, multi-sensory.

---

## Course 1: Wonder Lab — Colors, Shapes & Patterns

| Field | Value |
|-------|-------|
| **Age Group** | 3–5 (Early Explorers) |
| **Type** | FREE |
| **Difficulty** | Beginner |
| **Duration** | 3 weeks (9 lessons, ~15 min each) |
| **Prerequisite** | None |
| **Course Promise** | Your child will discover patterns, sorting, and shapes through playful exploration — building the foundations for math and logical thinking. |
| **Parent-Facing Value** | This course builds the observation and classification skills that research shows are essential for early mathematical reasoning. Your child will learn to notice, compare, and organize — the first steps toward problem-solving. |
| **Completion Criteria** | Complete all 9 lessons + final Shape Art project |
| **Badge** | "Pattern Explorer" — awarded on completion |
| **Certificate** | "Early Explorer: Colors, Shapes & Patterns" |

### Learner Outcomes
1. Compare objects by size, color, and shape
2. Sort items into groups using one or two attributes
3. Identify and extend simple repeating patterns (AB, ABB, ABC)
4. Recognize basic shapes in the environment
5. Create original artwork using shapes and patterns

### Module 1: See and Sort (Lessons 1–3)

**Lesson 1: Big, Small, Same, Different**
- Objective: Compare objects using size and visual attributes
- Summary: Children look at pairs of objects and decide — is it big or small? Same or different? They practice using comparison words.
- Activity: "Sorting Tray" — Sort 10 picture cards into Big/Small or Same/Different piles
- Assessment: Can the child correctly sort 7 of 10 cards

**Lesson 2: Sorting Treasure Baskets**
- Objective: Group objects by a chosen attribute (color, shape, or type)
- Summary: Children receive a mixed "treasure basket" of items and choose how to sort them into groups.
- Activity: Sort virtual items by color, then re-sort by shape, then by size
- Assessment: Child creates at least 2 different groupings from the same set

**Lesson 3: Sorting Challenge**
- Objective: Apply sorting skills to a new context
- Summary: Children sort animals by habitat, foods by color, and vehicles by type. They explain why they grouped things together.
- Activity: "My Sorting Rules" — draw or drag items into groups and tell the rule
- Assessment: Child can name the sorting rule they used

### Module 2: Pattern Play (Lessons 4–6)

**Lesson 4: What Comes Next?**
- Objective: Identify and extend a simple AB pattern
- Summary: Red-blue-red-blue... what comes next? Children learn that patterns repeat and that they can predict what comes next.
- Activity: Complete 5 pattern strips (color, shape, and sound patterns)
- Assessment: Child extends 4 of 5 patterns correctly

**Lesson 5: Make Your Own Pattern Train**
- Objective: Create an original repeating pattern
- Summary: Using colored blocks or stickers, children build their own "pattern train" — a line of repeating items they designed themselves.
- Activity: Build a pattern train with at least 3 full repeats
- Assessment: Pattern repeats correctly at least 3 times

**Lesson 6: Pattern Hunt**
- Objective: Recognize patterns in everyday life
- Summary: Patterns are everywhere — stripes on clothes, tiles on floors, petals on flowers. Children go on a pattern hunt.
- Activity: Find and photograph/draw 3 patterns in your home or outside
- Assessment: Child identifies at least 2 real-world patterns

### Module 3: Shapes in the World (Lessons 7–9)

**Lesson 7: Circle Hunt, Square Hunt**
- Objective: Identify circles, squares, triangles, and rectangles
- Summary: Children learn the names and features of 4 basic shapes, then hunt for them in their environment.
- Activity: "Shape Spy" — tap/click on shapes hidden in a scene illustration
- Assessment: Child identifies 4 basic shapes correctly

**Lesson 8: Shape Builders**
- Objective: Construct shapes using different materials
- Summary: Can you make a triangle with sticks? A circle with string? Children build shapes with craft materials.
- Activity: Build each shape using craft sticks, string, or playdough
- Assessment: Child constructs at least 3 different shapes

**Lesson 9: Shape Art (Final Project)**
- Objective: Create a picture using only shapes
- Summary: Children combine circles, squares, triangles, and rectangles to create a house, animal, robot, or any picture they imagine.
- Activity: "Shape Art Gallery" — create a picture using at least 5 shapes, name the shapes used
- Assessment: Completed artwork using at least 4 different shape types + verbal/written description

## Course 2: Little Coders Unplugged

| Field | Value |
|-------|-------|
| **Age Group** | 3–5 (Early Explorers) |
| **Type** | PAID |
| **Difficulty** | Beginner |
| **Duration** | 3 weeks (9 lessons, ~15 min each) |
| **Prerequisite** | Wonder Lab (recommended, not required) |
| **Course Promise** | Your child will learn to think in steps, give clear instructions, and solve simple problems — all the foundations of coding, without any screens. |
| **Parent-Facing Value** | Sequencing and algorithmic thinking are core skills for future coding success. This course builds them through movement, games, and storytelling — no screen time required for most activities. |
| **Completion Criteria** | Complete all 9 lessons + final "Robot Commander" challenge |
| **Badge** | "Step Master" — awarded on completion |
| **Certificate** | "Early Explorer: Unplugged Coding" |

### Learner Outcomes
1. Arrange actions in the correct order (first, next, last)
2. Give step-by-step instructions to reach a goal
3. Follow a simple sequence of directional commands
4. Identify when a sequence is wrong and suggest a fix
5. Create a simple step-by-step plan for a task

### Module 1: First Steps (Lessons 1–3)

**Lesson 1: First, Next, Last**
- Objective: Understand the concept of ordered steps
- Summary: Making a sandwich has steps. Getting dressed has steps. Children learn that order matters.
- Activity: Arrange 3–4 picture cards in the right order (e.g., wake up → brush teeth → eat breakfast)
- Assessment: Child correctly orders 3 sequences

**Lesson 2: Follow the Path**
- Objective: Follow a set of directional instructions
- Summary: Children follow a simple path on a grid — forward, turn left, turn right. They practice listening and moving.
- Activity: Floor grid walk — follow 5-step instruction cards to reach a treasure
- Assessment: Child follows a 5-step path correctly

**Lesson 3: My First Algorithm**
- Objective: Create a set of ordered instructions
- Summary: Children write (or draw) their own set of steps for a simple task — like making a paper airplane or brushing teeth.
- Activity: Draw a 4-step instruction card for a daily routine
- Assessment: Steps are in a logical order and the task would work if followed

### Module 2: Commands and Directions (Lessons 4–6)

**Lesson 4: Move Like a Robot**
- Objective: Follow exact commands without interpretation
- Summary: Robots do exactly what they are told — nothing more, nothing less. One child is the "robot" and the other gives commands.
- Activity: Robot movement game — command a partner with "step forward," "turn left," "pick up"
- Assessment: Successful 6-command sequence without errors

**Lesson 5: Tell a Friend What to Do**
- Objective: Give precise instructions to another person
- Summary: Can you guide your friend to draw a picture? Without showing them? Children practice giving clear, step-by-step commands.
- Activity: Blind drawing — instruct a partner to draw a simple shape or picture using only words
- Assessment: Partner's drawing matches the intended picture

**Lesson 6: Code the Adventure**
- Objective: Plan a sequence of moves to complete a challenge
- Summary: Children plan a route through a simple maze using directional command cards.
- Activity: Maze challenge — place arrow cards in order to guide a character from start to finish
- Assessment: Character reaches the goal in correct sequence

### Module 3: Fix the Mix-Up (Lessons 7–9)

**Lesson 7: Oops, That Didn't Work**
- Objective: Identify an error in a sequence
- Summary: Sometimes the steps are wrong — the robot goes the wrong way, the recipe makes a mess. Children learn to spot the "bug."
- Activity: Find the wrong step in 4 broken sequences and point to it
- Assessment: Child identifies the error in 3 of 4 sequences

**Lesson 8: Try Again**
- Objective: Fix a broken sequence and retry
- Summary: When something goes wrong, coders don't give up — they debug. Children fix the broken sequences from lesson 7.
- Activity: Swap, remove, or add steps to fix 3 broken sequences
- Assessment: Child successfully fixes at least 2 sequences

**Lesson 9: Robot Commander (Final Project)**
- Objective: Design and execute a complete command sequence
- Summary: Children create a 6+ step sequence to guide a character through a mini obstacle course. They test, debug, and complete the mission.
- Activity: Design a path through an obstacle scene using 6–8 command cards, then test it
- Assessment: Complete working sequence of 6+ steps that reaches the goal

## Course 3: Tiny Engineers

| Field | Value |
|-------|-------|
| **Age Group** | 3–5 (Early Explorers) |
| **Type** | PAID |
| **Difficulty** | Beginner |
| **Duration** | 3 weeks (9 lessons, ~15 min each) |
| **Prerequisite** | None |
| **Course Promise** | Your child will build towers, bridges, ramps, and moving structures — discovering how things stand up, fall down, and move. |
| **Parent-Facing Value** | Engineering play develops spatial reasoning, persistence, and problem-solving. Your child will learn to test ideas, handle failure, and improve designs — skills that matter in every subject. |
| **Completion Criteria** | Complete all 9 lessons + final "Super Structure" build |
| **Badge** | "Tiny Builder" — awarded on completion |
| **Certificate** | "Early Explorer: Engineering Play" |

### Learner Outcomes
1. Build a structure that stands on its own
2. Test whether a structure is strong or weak
3. Explain what makes a bridge or tower stronger
4. Build a ramp and predict how objects move on it
5. Improve a design after testing it

### Module 1: Build Up (Lessons 1–3)

**Lesson 1: Strong or Wobbly?**
- Objective: Test structures for stability
- Summary: Children build towers with blocks and test — can it stand? Can it hold a toy on top? They learn that some shapes and arrangements are stronger.
- Activity: Build 3 different towers and test which holds a small toy
- Assessment: Child identifies which tower is strongest and why

**Lesson 2: Bridge Builders**
- Objective: Build a bridge that spans a gap
- Summary: Using cardboard, blocks, or paper, children build a bridge between two stacks. Can it hold a toy car?
- Activity: Build a bridge that spans a 20cm gap and holds a small object
- Assessment: Bridge holds weight for 5 seconds

**Lesson 3: Tall Tower Challenge**
- Objective: Build the tallest possible structure with limited materials
- Summary: Children receive a fixed set of materials (10 blocks, paper cups, cardboard strips) and compete to build the tallest free-standing tower.
- Activity: Tallest tower challenge — measure height, test stability
- Assessment: Tower stands for 10 seconds without falling

### Module 2: Push, Pull, Roll (Lessons 4–6)

**Lesson 4: Ramp Races**
- Objective: Understand that ramp angle affects speed
- Summary: Children build ramps at different heights and race toy cars down them. Steeper = faster. They predict and test.
- Activity: Build 3 ramps at different angles, race a car, record which is fastest
- Assessment: Child correctly predicts that steeper ramps produce faster rolls

**Lesson 5: Wheels at Work**
- Objective: Explore how wheels reduce friction and help things move
- Summary: What moves easier — a box or a box on wheels? Children test objects with and without wheels.
- Activity: Push objects on carpet vs smooth floor, with wheels vs without
- Assessment: Child explains that wheels make things easier to move

**Lesson 6: Push and Pull Machines**
- Objective: Identify push and pull forces in everyday life
- Summary: Doors push and pull. Wagons pull. Swings push. Children identify forces in their world.
- Activity: Sort 10 picture cards into "push" and "pull" categories
- Assessment: Child correctly sorts at least 8 of 10 cards

### Module 3: Test and Improve (Lessons 7–9)

**Lesson 7: Can It Hold It?**
- Objective: Test a structure's load-bearing capacity
- Summary: Children build a small table or platform and test how many coins/blocks it can hold before collapsing.
- Activity: Build, predict the number it will hold, test, record result
- Assessment: Child records a prediction and actual result

**Lesson 8: Make It Better**
- Objective: Improve a failed design based on what was learned
- Summary: The structure from Lesson 7 probably collapsed. Children analyze why and rebuild a stronger version.
- Activity: Identify the weakness, redesign, rebuild, and retest
- Assessment: Version 2 holds more weight than version 1

**Lesson 9: Super Structure (Final Project)**
- Objective: Design, build, test, and present a structure
- Summary: Children pick a challenge (tallest tower, strongest bridge, or fastest ramp), build it, test it, and present it to family.
- Activity: Full design-build-test-present cycle with photos or video
- Assessment: Completed build + simple verbal explanation of what works and why

## Course 4: Space & Sky for Little Explorers

| Field | Value |
|-------|-------|
| **Age Group** | 3–5 (Early Explorers) |
| **Type** | PAID |
| **Difficulty** | Beginner |
| **Duration** | 3 weeks (9 lessons, ~15 min each) |
| **Prerequisite** | None |
| **Course Promise** | Your child will discover the Sun, Moon, stars, shadows, and rockets — sparking a lifelong wonder about the sky and space. |
| **Parent-Facing Value** | Space is the ultimate curiosity driver for young learners. This course channels that wonder into real observation skills, simple science, and creative play — all grounded in age-appropriate NASA and ESA educational resources. |
| **Completion Criteria** | Complete all 9 lessons + Straw Rocket launch project |
| **Badge** | "Sky Watcher" — awarded on completion |
| **Certificate** | "Early Explorer: Space & Sky" |

### Learner Outcomes
1. Describe the difference between day and night
2. Name the Sun, Moon, and stars and describe what they look like
3. Observe that the Moon appears to change shape
4. Explore how shadows form and change
5. Build and launch a simple straw rocket

### Module 1: Sky Watchers (Lessons 1–3)

**Lesson 1: Day and Night**
- Objective: Understand that Earth has daytime and nighttime
- Summary: When it's light, we see the Sun. When it's dark, we see the Moon and stars. Children explore what happens in daytime vs nighttime.
- Activity: Sort picture cards into "daytime activities" and "nighttime activities"
- Assessment: Child correctly sorts 8 of 10 cards

**Lesson 2: Sun, Moon, Stars**
- Objective: Identify the Sun, Moon, and stars in the sky
- Summary: The Sun gives us light and warmth. The Moon shines at night. Stars are very far away. Children observe and draw what they see.
- Activity: Draw or create a day sky picture and a night sky picture
- Assessment: Both pictures include correct elements (Sun in day, Moon and stars at night)

**Lesson 3: What's in the Sky Today?**
- Objective: Practice observing the sky and describing what is seen
- Summary: Children look at the real sky (or photographs) and describe what they see — clouds, colors, Sun position.
- Activity: Sky journal — draw or describe the sky at different times of day
- Assessment: Child records at least 2 sky observations

### Module 2: Moon and Motion (Lessons 4–6)

**Lesson 4: The Moon Changes**
- Objective: Observe that the Moon appears different on different nights
- Summary: Sometimes the Moon is round, sometimes it's a sliver. Children learn that the Moon changes shape over time (phases simplified).
- Activity: Match Moon shape pictures to their names (full, half, crescent)
- Assessment: Child identifies 3 Moon shapes

**Lesson 5: Shadow Play**
- Objective: Understand that shadows form when light is blocked
- Summary: Stand in the Sun — there's your shadow! Move and it moves. Children play with shadows and discover how they work.
- Activity: Shadow tracing outdoors or with a flashlight — trace your shadow at two different times
- Assessment: Child observes that shadows change size/direction

**Lesson 6: Light and Dark Experiments**
- Objective: Explore which objects make shadows and which let light through
- Summary: Some things block light (opaque), some let it through (transparent). Children test objects with a flashlight.
- Activity: Sort objects into "makes a shadow" and "light goes through"
- Assessment: Child correctly categorizes at least 6 of 8 objects

### Module 3: Rocket Time (Lessons 7–9)

**Lesson 7: What Is a Rocket?**
- Objective: Understand that rockets carry things into space
- Summary: Rockets push very hard to fly up into the sky. Astronauts ride in rockets. Children watch simple rocket launch videos (NASA Kids' Club level).
- Activity: Label a simple rocket diagram (nose, body, fins, engine)
- Assessment: Child labels 3 of 4 parts correctly

**Lesson 8: Countdown and Launch**
- Objective: Understand the concept of a launch sequence
- Summary: 5... 4... 3... 2... 1... Blast off! Children learn that launching a rocket follows a sequence — just like coding.
- Activity: Create a countdown sequence with body movements (crouch, count, jump)
- Assessment: Child performs the full countdown sequence

**Lesson 9: Make a Straw Rocket (Final Project)**
- Objective: Build and launch a simple paper straw rocket
- Summary: Using paper, tape, and a straw, children build a small rocket and launch it by blowing through the straw.
- Activity: Build straw rocket, decorate it, launch it, measure how far it flies
- Assessment: Rocket flies, child describes what they did

## Course 5: Creative Robot Stories

| Field | Value |
|-------|-------|
| **Age Group** | 3–5 (Early Explorers) |
| **Type** | PAID |
| **Difficulty** | Beginner |
| **Duration** | 3 weeks (9 lessons, ~15 min each) |
| **Prerequisite** | None (Wonder Lab recommended) |
| **Course Promise** | Your child will use storytelling and pretend play to understand what robots are, how they follow instructions, and how humans are different from machines. |
| **Parent-Facing Value** | This course builds early AI/robot literacy through safe, imaginative play. Children learn that robots follow instructions made by people — building a healthy, accurate understanding before they encounter real technology. |
| **Completion Criteria** | Complete all 9 lessons + Box Robot craft and story |
| **Badge** | "Robot Friend Maker" — awarded on completion |
| **Certificate** | "Early Explorer: Creative Robot Stories" |

### Learner Outcomes
1. Describe what a robot is in simple terms
2. Explain that robots follow instructions from people
3. Compare what robots can and cannot do vs humans
4. Create a simple robot character and story
5. Build a robot craft from household materials

### Module 1: Meet the Robot (Lessons 1–3)

**Lesson 1: What Is a Robot?**
- Objective: Understand that a robot is a machine that follows instructions
- Summary: Robots are machines made by people. They can move, sense, and do tasks — but only when someone tells them what to do.
- Activity: Watch short clips of real robots (warehouse bots, Mars rovers), then draw "my robot idea"
- Assessment: Child can say "a robot follows instructions"

**Lesson 2: Robot Feelings, Human Feelings**
- Objective: Distinguish between what robots and humans can do
- Summary: Robots can lift heavy things and work all day. But can they feel happy? Sad? Hungry? Children explore the difference.
- Activity: Sort cards into "robots can" and "only humans can" (lift heavy boxes vs feel sad)
- Assessment: Child correctly sorts at least 6 of 8 cards

**Lesson 3: Robots Need Instructions**
- Objective: Understand that robots cannot think on their own
- Summary: If you don't tell a robot what to do, it does nothing. Children practice being the "programmer" and the "robot."
- Activity: Robot-programmer partner game — one child gives instructions, the other follows exactly
- Assessment: Child understands that the robot only does what it's told

### Module 2: Story Commands (Lessons 4–6)

**Lesson 4: Robot Goes to School**
- Objective: Imagine a robot in a familiar setting and identify what it would need to be told
- Summary: If a robot went to school, what instructions would it need? Open the door. Sit down. Listen to the teacher. Children create a sequence.
- Activity: Create a 5-step "school day" instruction sequence for a robot
- Assessment: Sequence includes at least 5 logical steps

**Lesson 5: Robot Helper Mission**
- Objective: Design instructions for a robot to complete a helpful task
- Summary: Can a robot help clean up? Deliver a message? Water a plant? Children design a "mission" with step-by-step instructions.
- Activity: Draw a 4-panel comic of a robot helper completing a mission
- Assessment: Comic shows a clear sequence with a beginning and end

**Lesson 6: When the Robot Gets Confused**
- Objective: Understand what happens when instructions are unclear
- Summary: "Go to the thing and do the stuff" — that's confusing! Children learn that instructions must be specific and clear.
- Activity: Give a "bad instruction" and a "good instruction" for the same task and compare results
- Assessment: Child can improve a vague instruction to make it specific

### Module 3: Build a Robot Friend (Lessons 7–9)

**Lesson 7: Box Robot Craft**
- Objective: Build a robot from recycled materials
- Summary: Using boxes, tubes, bottle caps, and tape, children build their own robot friend.
- Activity: Build a robot from household materials, name it, describe what it does
- Assessment: Completed robot build with a name and purpose

**Lesson 8: My Robot's Story**
- Objective: Create a short story featuring the robot character
- Summary: Every robot has a story — where did it come from? What is its mission? Children tell the story of their robot.
- Activity: Tell, draw, or record a story about the robot — 3 parts: who, what, adventure
- Assessment: Story has a character, a task, and an ending

**Lesson 9: Robot Show and Tell (Final Project)**
- Objective: Present the robot build and story to others
- Summary: Children present their box robot and story to family or classmates — sharing what they made and what they learned about robots.
- Activity: Show and tell presentation (live, recorded video, or drawn poster)
- Assessment: Child presents the robot and explains at least 2 things robots do
