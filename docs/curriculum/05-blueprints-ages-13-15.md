### Learner Outcomes
1. Explain the role of HTML for structure and CSS for styling in a webpage
2. Build a semantic HTML page with headings, lists, links, and images
3. Apply CSS to control colors, fonts, spacing, and layout
4. Create a multi-section layout using CSS flexbox or basic grid
5. Publish a personal project page online

### Module 1: Page Structure (Lessons 1–3)

**Lesson 1: HTML — Content and Meaning**
- Objective: Understand that HTML provides structure and semantic meaning to content
- Summary: HTML is the skeleton of every webpage. Tags tell the browser what things are — heading, paragraph, list, image. Children write their first HTML file from scratch in a code editor.
- Activity: Create an HTML file with: DOCTYPE, html, head, body, title, one h1, two p tags, and view it in a browser
- Assessment: File opens in browser, all elements render correctly, page has a visible title in the tab

**Lesson 2: Headings, Lists, Images, Links**
- Objective: Use h1–h3, ul/ol/li, img, and a tags to build a structured content page
- Summary: A real webpage has sections, lists of things, pictures, and links. Children build a content-rich page about a topic they choose — using all 4 key element types.
- Activity: Build a single-topic page (e.g., favorite sport, animal, or STEM topic) using h1, h2, ul, img, and 2 external links
- Assessment: All 5 element types used correctly; links open correctly; image displays

**Lesson 3: Semantic HTML**
- Objective: Use semantic elements (header, nav, main, section, footer) to structure a page
- Summary: Modern HTML uses meaningful elements — not just divs. header, nav, main, section, footer tell the browser (and screen readers) what each area does. Children refactor their page to use semantic structure.
- Activity: Restructure the page from Lesson 2 using semantic elements; add a nav with 3 links and a footer
- Assessment: Page uses at least 5 semantic elements; nav and footer are present and functional

### Module 2: Design and Style (Lessons 4–6)

**Lesson 4: CSS — Colors, Fonts, Layout**
- Objective: Apply CSS to change colors, font families, and basic spacing
- Summary: CSS is the paint and design of the web. Children write their first CSS file, link it to HTML, and apply color, font, and spacing changes to their page.
- Activity: Create a linked CSS file; set background-color, color, font-family for body; set different colors for h1 and h2
- Assessment: CSS file is linked, 3+ style rules applied, visual changes visible in browser

**Lesson 5: Make It Readable and Clear**
- Objective: Apply CSS principles of readability: line-height, max-width, padding, margin
- Summary: Good design isn't decoration — it's clarity. Long lines are hard to read. Tight spacing is exhausting. Children apply readability best practices to their page.
- Activity: Set max-width on the main content area, add padding to sections, adjust line-height and font-size for body text
- Assessment: Page is visually readable; line length, spacing, and font size are appropriate

**Lesson 6: Layout with Flexbox**
- Objective: Use CSS flexbox to create a 2-column or card-based layout
- Summary: Flexbox is the most practical CSS layout tool. Children use it to create a 2-column section or a row of cards — the layouts that appear on almost every modern website.
- Activity: Create a "projects" section using flexbox that displays 3 project cards side by side
- Assessment: 3 cards display in a row using flexbox; cards have equal width and spacing

### Module 3: Interactive Thinking (Lessons 7–9)

**Lesson 7: Simple JavaScript Ideas**
- Objective: Add one JavaScript interaction to a page (button click, show/hide, alert)
- Summary: JavaScript makes pages interactive. Children write their first JS — a button that changes the background color, shows a message, or toggles text — linking behavior to user actions.
- Activity: Add a button that changes the page background color when clicked using JavaScript
- Assessment: Button exists, click event fires, background color changes

**Lesson 8: Forms and User Input**
- Objective: Create an HTML form with input fields and a submit button
- Summary: Forms are how users talk to websites. Children build a contact form or quiz form with text inputs, a select dropdown, and a submit button — understanding how forms collect data.
- Activity: Build a 4-field contact form: name, email, subject (dropdown), message. Add basic CSS styling.
- Assessment: Form renders correctly with all 4 field types; submit button is present; form is styled

**Lesson 9: Personal Project Page (Final Project)**
- Objective: Build and publish a complete personal project page
- Summary: Children build a personal page showcasing who they are, their interests, and their work — using all the HTML, CSS, and JavaScript skills from the course. They publish it using GitHub Pages or a similar free host.
- Activity: Build a 3-section personal page: About Me / Projects / Contact Form. Publish online.
- Assessment: Published URL is accessible; page has 3 sections, semantic HTML, custom CSS, at least 1 interactive element
