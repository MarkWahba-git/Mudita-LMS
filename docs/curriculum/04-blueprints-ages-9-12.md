### Learner Outcomes
1. Build an animated interactive story using events, motion, and costumes
2. Implement a score variable that updates during gameplay
3. Add win/lose conditions and multiple levels to a game
4. Debug a broken Scratch project using systematic testing
5. Publish a complete, playable, polished Scratch game

### Module 1: Interactive Stories (Lessons 1–3)

**Lesson 1: Events and Motion**
- Objective: Use event blocks (green flag, key press, sprite click) to trigger motion
- Summary: In Scratch, nothing happens until an event triggers it. Children learn the 3 key event blocks and use them to make sprites move, appear, and react — the building blocks of any interactive project.
- Activity: Build a scene where clicking the sprite makes it move and play a sound; pressing spacebar changes the background
- Assessment: At least 2 different events trigger 2 different actions correctly

**Lesson 2: Say, Think, Change Scene**
- Objective: Create dialogue using "say" blocks and scene changes using backdrop switches
- Summary: Stories have characters who talk and scenes that change. Children add speech bubbles, thought bubbles, and backdrop switches to create a story with at least 2 scenes and 2 characters.
- Activity: Build a 2-scene mini story with at least 2 sprites talking using "say for X seconds" blocks
- Assessment: Story has 2 scenes, 2 speaking characters, logical sequence

**Lesson 3: My Animated Story**
- Objective: Create an original animated story with beginning, middle, and end
- Summary: Combining events, motion, dialogue, and scene changes, children build a complete original story — their own plot, their own characters.
- Activity: Build a 3-scene animated story (introduction → problem → resolution) with at least 3 sprites
- Assessment: Story has a clear beginning, middle, and end; all animations run in correct order

### Module 2: Game Mechanics (Lessons 4–6)

**Lesson 4: Scores and Variables**
- Objective: Create a variable to track and display a score during gameplay
- Summary: Variables are containers that store information. A score variable starts at 0 and goes up when something good happens. Children create a score counter linked to in-game events.
- Activity: Build a catching game where clicking a falling object adds 1 to the score
- Assessment: Score variable is created, displayed on screen, and updates correctly when the event triggers

**Lesson 5: Levels, Timers, and Win Conditions**
- Objective: Add a timer, a level-up mechanic, and a win/lose condition
- Summary: Great games have structure — time limits, difficulty increases, and clear win/lose states. Children add these layers to their catching game.
- Activity: Add a 30-second timer that ends the game, a "you win" message when score reaches 10, and a "time's up" message
- Assessment: Timer counts down, win condition triggers at correct score, game ends properly

**Lesson 6: Clones and Enemies**
- Objective: Use the clone block to create multiple instances of a sprite
- Summary: Clones let one sprite become many — enemies, coins, obstacles — without creating separate sprites. Children add clone-based enemies or collectibles to make their game more dynamic.
- Activity: Use the "create clone" block to spawn 5 moving obstacles that the player must avoid
- Assessment: Clones spawn, move independently, and cause a correct game reaction on touch

### Module 3: Debug and Publish (Lessons 7–9)

**Lesson 7: Testing with Friends**
- Objective: Conduct user testing and identify at least 3 areas to improve
- Summary: Game designers test with real players — and the feedback is often surprising. Children swap projects with a classmate (or family member) and watch them play, noting confusion and frustration points.
- Activity: 10-minute user test session — observer watches without helping, fills in a 3-question feedback form
- Assessment: Feedback form completed; child identifies at least 3 specific issues

**Lesson 8: Improve the Player Experience**
- Objective: Address feedback by fixing bugs, improving instructions, and polishing presentation
- Summary: Children take their top 3 feedback items and fix them — clearer instructions, smoother controls, better visuals. This is professional-level iteration.
- Activity: Fix at least 3 identified issues and add a title screen with game instructions
- Assessment: Title screen added, 3 fixes implemented, game plays more smoothly than Version 1

**Lesson 9: Publish Your Game (Final Project)**
- Objective: Complete, polish, name, describe, and publish a finished Scratch game
- Summary: Children finalize their game, write a description in Scratch's "Instructions" and "Notes" fields, and publish it. This is their first public creative work — something they can share with pride.
- Activity: Publish game to Scratch with title, description, instructions, and custom thumbnail
- Assessment: Game is published, all required fields are complete, game is fully playable by a first-time player
